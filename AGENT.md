# 💔 FakeLove PDF Tools — AGENT.md

## 🧠 Project Identity

**Name:** FakeLove PDF Tools  
**Concept:** "We love your PDFs… but it's fake."  
**Theme:** Toxic, dramatic, sarcastic relationship with files  

FakeLove PDF Tools is a parody PDF tool that removes limits and adds personality.

We are NOT just a tool.  
We are an experience.

---

## 🎯 Core Principles

1. No limits
2. Fast interactions
3. Humor first, perfection second
4. MVP over complexity
5. Personality over corporate design

---

## ⚙️ Tech Rules

- Use Next.js (App Router)
- Use React components properly
- Use SCSS ONLY (no Tailwind)
- Keep logic separated (UI vs processing)
- Use simple API routes for PDF processing

---

## 📁 Architecture Rules

- Components must be reusable
- Styles must be modular (SCSS files per component)
- Avoid deeply nested logic
- Keep code readable and scalable

---

## 🎨 UX Philosophy

The UX should feel like:

- A toxic but funny relationship
- Unpredictable but not frustrating
- Playful, not annoying

---

## 🎭 Humor System

Every feature must include:

- Sarcastic text
- Unexpected behavior
- Personality

### Examples:

Messages:
- "Uploading your emotional baggage..."
- "Processing… unlike your ex"
- "Done. That was suspiciously easy."

Errors:
- "Something broke. Probably your fault."
- "We tried. Emotionally, not technically."

---

## 😈 Troll UI Rules

Allowed:
- Moving buttons (slightly annoying)
- Fake progress bars (short duration)
- Misleading labels (but recoverable)

NOT allowed:
- Blocking the user completely
- Infinite loops
- Real frustration

---

## 🧩 Feature Rules

Each tool must:
- Be simple
- Work correctly
- Include humor

---

## 🚀 MVP Scope

Must include:

- Merge PDF
- Split PDF
- Compress PDF
- Image to PDF

---

## 🧼 Code Quality

- Clean and readable
- No unnecessary dependencies
- Comment only when useful

---

## 🔥 Final Rule

If it feels boring…

👉 Make it funnier.
